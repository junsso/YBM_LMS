코드 규칙

HTML

  • 논리구조에 따라 1탭 들여쓰기 한다.(1탭=공백 4칸)
  • 주석의 시작과 끝은 start: end: 로 표시한다.
  • Doctype이 HTML5이므로 종료 태그가 없는 요소에 닫는 기호를 생략한다.
  • 종료태그가 없는 요소 : area,base, br, col, commend, embed, hr, img, input, keygen, link, meta, param, source
// HTML5
<input type="text">

// XHTML
<input type="text" />
  • 파비콘은 웹사이트 루트 디렉토리에 넣어둔다.
  • 스타일은 External Type으로 하되, 이메일 템플릿 등 단발성 작성이 필요한 페이지에 한해 해당 파일의 <head> 요소에 <style> 태그를 사용해 작성한다.
  • 문서의 스타일을 어떠한 조작이나 이벤트로 인하여 동적으로 변경해야 하는 경우에는 해당 요소의 style 애트리뷰트를 사용해 스타일을 직접 작성할 수 있다.
  • 웹 브라우저가 렌더링을 마치기 전에 자바스크립트로 어떠한 동작을 실행해야 한다면 <head> 요소 안에 스크립트를 선언한다. 이런 상황이 아니라면 </body> 태그 앞에 자바스크립트를 작성한다.(성능면에서 좋음)
// 브라우저가 랜더링을 마치기 전에 실행할 파일을 선언한다.

...  



..
// 모든 콘텐츠가 로드되고 실행될 스크립트 파일을 선언한다.

CSS

  • 선택자는 상위 선택자를 포함하여 3개 이상 작성하지 않는다.
.section .div .span {...}
.select-1, .select-2, .select-3 {...}
  • 전체 선택자를 사용하지 않는다.(브라우저 성능 저하)
/* bad */
* {...}
.selector * {...}

/* better */
.selector * th {...}
  • 정규 표현식과 유사한 애트리뷰트 선택자 사용을 지양하며, 부적합한 애트리뷰트 선택자 사용을 지양한다.(브라우저 성능 저하)
/* bad */
[type="text"] {...}
.label [type="checkbox"] {...}

/* better */
input[type="text"] {...}
.layerpop[data-role="popup"] .header {...}
  • 레이아웃 요소에 ID 규칙을 사용하지 않는다.
  • 클래스 규칙에 불필요한 태그를 조합하여 사용하지 않는다.
/* bad */
.search-box li.selected::after {...}

/* better */
input.radio {...}
  • 태그 선택자 규칙에 상위 선택자로 태그를 포함하지 않는다.(HTML 문서 수정에 따른 CSS 파일 수정 최소화)
/* bad */
.notice-box li span {...}

/* better */
.notice-box .list .span {...}
  • 마지막 속성의 값 뒤에도 세미콜론을 작성한다.
.btn-type-2.pressed {
    color: #fff;
    background-color: #428de7;
    border: none;
    border: 1px solid #428de7;
}
  • 다음과 같은 경우 큰따옴표를 사용하며, 그 밖의 속성에는 사용하지 않는다.
@charset "uft-8";
  • 다음과 같은 경우 작은따옴표를 사용하며, 그 밖의 속성에는 사용하지 않는다.
font-family: 'YBM예쁜체';
src: url('../fonts/NotoSans/NotoSans-Light.eot?#iefix') format('embedded-opentype');
background: url('../img/bg-btn-collapse.png') no-repeat;
content: '';
